enum Response(value: Byte) : Enum<Response>
Content copied to clipboard

Response form has 2 types: Response field -> | Response | Body field -> | BODY | | Response | BODY | 1 4

| Response | BODY | 1 n -> n represents variables length of body, this is crucial when writing sensor's data into client's input stream

each response contains of 5 bytes 1^st byte represents response type following 4 bytes represent additional response data. Some responses will use this space, some will leave it empty. For example: Response type: CONFIGURE_Y CONFIGURE_Y.byteValue, -, -, -, - -> informs that it can get number of bytes requested in CONFIGURE request. Response type: IS_CONNECTED_Y IS_CONNECTED_Y.byteValue, -, -, -, - Response type: START_READING_Y START_READING_Y.byteValue, 0, 0, 0, 0b0000_0100 -> informs that reading started and that each time it will send 4 bytes Response type: READING READING.byteValue, x, x, x, x, ... , n^th 'x' byte etc.
